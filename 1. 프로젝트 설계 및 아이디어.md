# 저점매수 프로젝트 설계 초안

## 1. 프로젝트 개요

### 1.1 프로젝트 목적

눌림워치는 사용자가 평소 관심을 가지고 있던 주식이 매수 검토 가능한 가격 조정 구간에 들어왔는지 판단하기 위한 개인용 주식 눌림 감시 대시보드이다.

이 서비스는 사용자가 관심 종목으로 지정한 주식에 대해 현재 가격 위치, 추세 유지 여부, 조정 깊이, 거래량 안정성, 반등 확인 여부를 종합하여 눌림점수를 제공한다.

핵심 목적은 사용자가 차트를 매번 직접 확인하지 않아도 관심 종목이 관찰 구간, 매수 검토 구간, 추세 훼손 구간 중 어디에 있는지 빠르게 판단하도록 돕는 것이다.

### 1.2 초기 개발 범위

초기 버전은 개인 사용 목적의 MVP로 개발한다.

초기 개발 범위는 다음과 같다.

1. 관심 종목 등록
2. 종목별 현재가 및 최근 가격 데이터 조회
3. 최근 고점 대비 하락률 계산
4. 이동평균선 기준 가격 위치 계산
5. RSI 계산
6. 거래량 변화 계산
7. 눌림점수 계산
8. 상태 분류
9. 종목별 간단한 해석 문구 제공
10. 웹 대시보드 화면 제공

초기 버전에서는 다음 기능을 제외한다.

1. 실시간 매수 추천
2. 자동 매매
3. 재무제표 기반 기업가치 분석
4. 뉴스 요약
5. AI 기반 종목 추천
6. 커뮤니티 기능
7. 유료 결제 기능

## 2. 핵심 사용자 시나리오

### 2.1 개인 투자자 시나리오

사용자는 평소 사고 싶었던 종목을 관심 목록에 등록한다.

서비스는 매일 또는 사용자가 접속할 때 관심 종목의 가격 데이터를 분석한다.

사용자는 각 종목의 눌림점수와 상태를 보고 현재 매수 검토할 만한 위치인지 판단한다.

예시는 다음과 같다.

```text
사용자는 구글, 엔비디아, 삼성전자, QQQ, QLD를 관심 종목으로 등록한다.

서비스는 각 종목의 현재가, 최근 고점 대비 하락률, 이동평균선 위치, RSI, 거래량 상태를 계산한다.

사용자는 눌림점수가 높은 종목만 우선 확인한다.

사용자는 눌림점수는 높지만 반등 확인이 부족한 종목은 관찰 목록에 둔다.

사용자는 추세 훼손으로 표시된 종목은 단순 눌림으로 판단하지 않는다.
```

## 3. 서비스 핵심 개념

### 3.1 눌림

눌림은 상승 추세가 유지되는 종목이 단기적으로 가격 조정을 받는 상태를 의미한다.

이 서비스에서 눌림은 단순히 가격이 많이 하락한 상태를 의미하지 않는다. 상승 추세 유지, 적정 조정 깊이, 이동평균선 근접, 거래량 안정, 반등 확인 가능성이 함께 고려되어야 한다.

### 3.2 눌림점수

눌림점수는 관심 종목이 현재 매수 검토 가능한 눌림 구간에 있는지를 100점 만점으로 표현한 점수이다.

눌림점수는 매수 추천 점수가 아니다. 눌림 후보로 관찰할 가치가 있는지를 판단하는 필터링 점수이다.

### 3.3 상태 분류

종목별 상태는 다음 중 하나로 분류한다.

| 상태 | 의미 |
| --- | --- |
| 관찰 전 | 아직 눌림 조건에 충분히 들어오지 않은 상태 |
| 눌림 진입 | 상승 추세 안에서 가격 조정이 시작된 상태 |
| 매수 검토 | 눌림 깊이와 추세 조건이 비교적 양호한 상태 |
| 반등 확인 필요 | 가격 위치는 괜찮으나 재상승 신호가 부족한 상태 |
| 추세 훼손 | 가격이 많이 하락했지만 눌림이 아니라 하락 전환일 가능성이 높은 상태 |

## 4. 눌림점수 산식 초안

### 4.1 총점 구조

```text
눌림점수 = 추세 유지 점수
         + 조정 깊이 점수
         + 이동평균선 근접도 점수
         + 거래량 안정도 점수
         + 반등 확인 점수
```

| 항목 | 배점 |
| --- | ---: |
| 추세 유지 | 35점 |
| 조정 깊이 | 25점 |
| 이동평균선 근접도 | 15점 |
| 거래량 안정도 | 15점 |
| 반등 확인 | 10점 |
| 총점 | 100점 |

### 4.2 추세 유지 점수

추세 유지 점수는 종목이 여전히 상승 추세 안에 있는지를 판단한다.

| 조건 | 점수 |
| --- | ---: |
| 현재가가 20일 이동평균선 위 | 10점 |
| 20일 이동평균선이 60일 이동평균선 위 | 10점 |
| 60일 이동평균선 기울기가 양수 | 10점 |
| 최근 저점이 이전 저점보다 높음 | 5점 |
| 최대 점수 | 35점 |

### 4.3 조정 깊이 점수

조정 깊이 점수는 최근 고점 대비 현재가가 얼마나 하락했는지 판단한다.

기준 고점은 기본적으로 최근 60거래일 고점으로 설정한다.

| 최근 60거래일 고점 대비 하락률 | 점수 | 해석 |
| --- | ---: | --- |
| 0퍼센트 이상 3퍼센트 미만 | 5점 | 아직 충분히 눌리지 않음 |
| 3퍼센트 이상 8퍼센트 이하 | 25점 | 일반적인 눌림 후보 |
| 8퍼센트 초과 15퍼센트 이하 | 18점 | 깊은 조정 후보 |
| 15퍼센트 초과 25퍼센트 이하 | 8점 | 추세 훼손 주의 |
| 25퍼센트 초과 | 0점 | 단순 눌림으로 보기 어려움 |

### 4.4 이동평균선 근접도 점수

이동평균선 근접도 점수는 현재가가 주요 이동평균선 근처에 위치하는지 판단한다.

| 조건 | 점수 |
| --- | ---: |
| 현재가가 20일선과 3퍼센트 이내 | 8점 |
| 현재가가 60일선과 5퍼센트 이내 | 7점 |
| 최대 점수 | 15점 |

단, 현재가가 60일선 아래로 크게 이탈한 경우에는 추세 훼손 가능성을 별도 판단한다.

### 4.5 거래량 안정도 점수

거래량 안정도 점수는 하락 과정에서 매도 압력이 과도한지 판단한다.

기준 거래량은 최근 20거래일 평균 거래량으로 설정한다.

| 조건 | 점수 |
| --- | ---: |
| 하락일 평균 거래량이 최근 20일 평균 거래량보다 낮음 | 8점 |
| 최근 3거래일 동안 급락 거래량이 없음 | 4점 |
| 반등일 거래량이 전일보다 증가 | 3점 |
| 최대 점수 | 15점 |

### 4.6 반등 확인 점수

반등 확인 점수는 눌림 이후 가격이 다시 위로 전환될 가능성이 있는지 판단한다.

| 조건 | 점수 |
| --- | ---: |
| 종가가 전일 고가 돌파 | 4점 |
| 종가가 5일 이동평균선 회복 | 3점 |
| RSI가 전일 대비 상승 | 2점 |
| 양봉 마감 | 1점 |
| 최대 점수 | 10점 |

## 6. 종목 해석 문구 생성 규칙

### 6.1 해석 문구 목적

해석 문구는 사용자가 점수의 이유를 빠르게 이해하도록 돕는다.

해석 문구는 매수 추천 문구가 아니어야 한다.

### 6.2 문구 예시

#### 6.2.1 매수 검토 상태

```text
현재 상승 추세는 유지되고 있으며, 최근 고점 대비 충분한 조정이 발생했습니다. 이동평균선 근처에서 가격이 안정되고 있어 매수 검토 구간으로 볼 수 있습니다. 다만 실제 진입 전에는 반등 확인과 손절 기준을 별도로 확인해야 합니다.
```

#### 6.2.2 눌림 진입 상태

```text
상승 추세 안에서 가격 조정이 시작된 상태입니다. 아직 반등 확인 신호는 충분하지 않으므로 바로 진입하기보다는 추가 가격 흐름을 관찰하는 것이 적절합니다.
```

#### 6.2.3 반등 확인 필요 상태

```text
가격은 조정 구간에 들어왔지만 반등 신호가 약합니다. 전일 고가 돌파, 5일선 회복, 거래량 증가 여부를 확인할 필요가 있습니다.
```

#### 6.2.4 추세 훼손 상태

```text
가격은 많이 내려왔지만 상승 추세가 약해진 상태입니다. 단순 눌림이라기보다는 하락 전환 가능성을 함께 봐야 합니다.
```

## 8. 데이터 소스 후보

### 8.1 국내 주식

국내 주식 데이터는 다음 후보 중 선택한다.

| 후보 | 장점 | 단점 |
| --- | --- | --- |
| pykrx | 국내 주식 데이터 접근이 비교적 쉬움 | 운영 환경 의존성 확인 필요 |
| FinanceDataReader | 국내와 해외 일부 데이터 접근 가능 | 데이터 안정성 확인 필요 |
| 증권사 Open API | 실시간성 및 안정성 가능 | 인증, 개발 복잡도 증가 |

### 8.2 미국 주식

미국 주식 데이터는 다음 후보 중 선택한다.

| 후보 | 장점 | 단점 |
| --- | --- | --- |
| yfinance | 구현이 쉬움 | 비공식 API 성격, 운영 안정성 주의 |
| Alpha Vantage | API 형태로 사용 가능 | 무료 한도 제한 |
| Polygon.io | 데이터 품질 우수 | 비용 발생 가능 |
| Twelve Data | API 형태로 사용 가능 | 무료 한도 제한 |

### 8.3 초기 권장 방향

초기 개인용 MVP에서는 yfinance 또는 FinanceDataReader 기반으로 빠르게 구현한다.

운영 안정성이 필요해지는 시점에 유료 API 또는 증권사 API를 검토한다.

## 9. 시스템 구성 초안

### 9.1 전체 구조

```text
사용자
  입력: 관심 종목 등록 및 조회
  출력: 눌림점수 대시보드 확인

프론트엔드
  관심 종목 목록 화면
  종목 상세 화면
  눌림점수 및 차트 표시

백엔드 API
  관심 종목 관리
  가격 데이터 조회
  기술 지표 계산
  눌림점수 계산
  상태 및 해석 문구 생성

데이터 저장소
  사용자 관심 종목
  종목 가격 데이터 캐시
  계산 결과 캐시
```

### 9.2 추천 기술 스택 초안

| 영역 | 후보 |
| --- | --- |
| 프론트엔드 | Next.js, React, Tailwind CSS, shadcn/ui |
| 백엔드 | Next.js API Routes 또는 FastAPI |
| 데이터베이스 | Supabase PostgreSQL |
| 배포 | Vercel |
| 배치 작업 | GitHub Actions, Vercel Cron, Supabase Scheduled Jobs 중 선택 |
| 차트 | Recharts, Lightweight Charts 중 선택 |

### 9.3 개인 프로젝트 기준 권장 스택

초기에는 다음 구성이 가장 단순하다.

```text
Next.js
React
Tailwind CSS
shadcn/ui
Supabase PostgreSQL
Vercel
외부 가격 데이터 API
```

단, Python 기반 데이터 수집과 계산이 편하면 백엔드 계산 모듈은 FastAPI 또는 별도 Python 배치로 분리할 수 있다.

## 10. 화면 설계 초안

### 10.1 관심 종목 대시보드

대시보드에는 관심 종목 목록과 핵심 점수를 표시한다.

| 표시 항목 | 설명 |
| --- | --- |
| 종목명 | 관심 종목 이름 |
| 현재가 | 최신 종가 또는 현재가 |
| 등락률 | 전일 대비 등락률 |

## 13. 계산 로직 의사코드

### 13.1 눌림점수 계산

```pseudo
function calculatePullbackScore(priceData):
    ma5 = movingAverage(close, 5)
    ma20 = movingAverage(close, 20)
    ma60 = movingAverage(close, 60)
    rsi14 = calculateRsi(close, 14)

    recentHigh60 = max(high over last 60 trading days)
    drawdown = (currentClose - recentHigh60) / recentHigh60 * 100

    trendScore = calculateTrendScore(currentClose, ma20, ma60)
    drawdownScore = calculateDrawdownScore(drawdown)
    maDistanceScore = calculateMaDistanceScore(currentClose, ma20, ma60)
    volumeScore = calculateVolumeScore(priceData)
    reboundScore = calculateReboundScore(priceData, ma5, rsi14)

    totalScore = trendScore + drawdownScore + maDistanceScore + volumeScore + reboundScore

    status = classifyStatus(totalScore, priceData, ma20, ma60, drawdown)
    summary = generateSummary(status, totalScore, drawdown, trendScore, reboundScore)

    return {
        totalScore,
        trendScore,
        drawdownScore,
        maDistanceScore,
        volumeScore,
        reboundScore,
        status,
        summary
    }
```

### 13.2 강제 추세 훼손 판단

```pseudo
function isTrendBroken(currentClose, ma20, ma60, drawdown, priceData):
    if currentClose < ma60 * 0.92:
        return true

    if drawdown < -25:
        return true

    if ma20 < ma60:
        return true

    if hasMultipleHighVolumeBearishCandles(priceData, lastDays = 5):
        return true

    return false
```
